# Bug hunt — 2026-04-18 (Caldera exercise against test001)

Bugs found during live exercise, tracked here for PR #146 or follow-up work.
Severity: **blocker** = prevents core workflow, **major** = feature looks broken,
**polish** = cosmetic.

## Already fixed in PR #146

1. ✅ **blocker** SIGMA / graph_crypto_miner incidents shipped `entities=[]` — Service
   entity type was dropped during graph ingestion. Threats tab pivots empty.
2. ✅ **major** Date picker in Threats tab stuck on stale URL query (`?date=YYYY-MM-DD`
   from a previous session) — defaulted to yesterday on reload.
3. ✅ **blocker** Pivot tabs (IP / User / Detector) hidden inside `display:none`
   wrapper in HTML. UI only ever rendered default IP pivot.
4. ✅ **blocker** Detector-pivot drill-down returned empty journey
   (`PivotKind::Detector => None` short-circuit in investigation.rs).

## Discovered in this hunt

5. 🚨 **blocker** Retroactive dismiss races AI triage.
   `loops/slow_loop.rs:572-616` dismisses every Incident node whose
   `decision.is_none()`. Intended for pre-noise-gate historical incidents, but
   kills current incidents before AI finishes its 30s+ inference — 47 of 61
   decisions today are `dismiss` with reason *"Retroactive dismiss: historical
   incident with no decision"*. Net effect: responder never fires on real
   threats. **Fix candidate**: only dismiss incidents older than N minutes
   (e.g. ≥ 15 min) to let the AI triage loop finish first.

6. ℹ️ **polish** `GET /api/live-feed/geoip` returns HTTP 400 when called with
   no `ips=` param. Not called by the frontend; smoke-test artefact only.
   Optionally return empty list on empty query for robustness.

7. ℹ️ **not a bug** `GET /api/graph/threats` returns empty because
   `threat_intel_hits()` requires IPs tagged by threat-intel feeds. Caldera
   attacks run from local processes with no external IPs, so there are no
   hits — this is correct behaviour.
